## Обзор проекта

**QR Generator** — однофайловое веб-приложение для генерации QR-кодов прямо в браузере.  
Поддерживает два режима: **Текст / URL** и **Контакт** (vCard 3.0 — телефон сканирует код и сразу предлагает добавить контакт).

Никакого серверного бэкенда, никакой сборки, никакого Node.js. Просто открой `index.html`.

## Режимы работы

Переключение между режимами — кнопки вкладок в верхней части карточки.

### Текст / URL

Введи любую строку: ссылку, текст, email (`mailto:...`), номер телефона (`tel:+7...`) и нажми **→ Создать QR-код** (или Enter).

### Контакт

Заполни поля контакта. Приложение сформирует строку vCard 3.0 и закодирует её в QR.  
Когда телефон отсканирует код — iOS и Android предложат сохранить контакт в адресную книгу.

---

## Контакт-режим (vCard)

### Поля формы

| Поле | Обязательно | Описание |
|---|---|---|
| Имя | — | Имя контакта |
| Фамилия | — | Фамилия контакта |
| Компания | — | Название организации |
| Должность | — | Должность / роль |
| Телефон | * | Мобильный номер (основной) |
| Доп. телефон | — | Рабочий или второй номер |
| Email | * | Электронная почта |
| Сайт | — | URL личного сайта или компании |
| Адрес | — | Рабочий адрес |
| Заметка | — | Произвольный текст |

\* Хотя бы одно из полей «Имя/Фамилия», «Телефон» или «Email» должно быть заполнено.

### Что происходит при сканировании

1. Камера телефона (или стороннее приложение) считывает QR
2. Устройство распознаёт формат **vCard 3.0**
3. Появляется диалог: *«Добавить контакт?»* с уже заполненными полями
4. Пользователь нажимает «Сохранить» — контакт в адресной книге

Протестировано на:
- iOS 16+ (камера, Wallet)
- Android 12+ (Google Lens, стандартная камера Samsung / Pixel)

### Пример сгенерированной vCard

```
BEGIN:VCARD
VERSION:3.0
N:Иванов;Иван;;;
FN:Иван Иванов
ORG:ООО Ромашка
TITLE:Менеджер по продажам
TEL;TYPE=CELL:+7 900 000 00 00
TEL;TYPE=WORK:+7 800 000 00 00
EMAIL:ivan@example.com
URL:https://example.com
ADR;TYPE=WORK:;;Москва, ул. Примерная, д. 1;;;;
NOTE:Любой текст
END:VCARD
```

## Параметры генерации

### Размер

| Значение | Рекомендуется для |
|---|---|
| 128px | Иконки, превью |
| 200px | Экран, небольшие макеты |
| 256px | Стандартное использование |
| 400px | Печать, плакаты |

> Контактный QR содержит больше данных, чем простая ссылка — для печати рекомендуется 256px и выше.

### Уровень коррекции ошибок

| Уровень | Восстановление | Когда использовать |
|---|---|---|
| **L** | ~7% | Цифровые экраны, идеальные условия |
| **M** | ~15% | Общий случай (по умолчанию) |
| **Q** | ~25% | Частично перекрытые коды |
| **H** | ~30% | Печать, нанесение на поверхности, логотип поверх QR |

### Цвета

- **Цвет кода** (`colorDark`) — цвет тёмных модулей, по умолчанию `#000000`
- **Фон** (`colorLight`) — цвет фона, по умолчанию `#ffffff`

> ⚠️ Соблюдай контраст: тёмный код на светлом фоне читается надёжнее.

## Как это работает

### Библиотека qrcodejs

```html
<script src="https://cdnjs.cloudflare.com/ajax/libs/qrcodejs/1.0.0/qrcode.min.js"></script>
```

Рисует QR в `<canvas>` внутри целевого `div`.

### Генерация QR (JS)

```javascript
new QRCode(document.getElementById('qr-output'), {
  text: vcardString,              // строка vCard или произвольный текст
  width: 256,
  height: 256,
  colorDark: '#000000',
  colorLight: '#ffffff',
  correctLevel: QRCode.CorrectLevel.M,
});
```

### Построение vCard

```javascript
function buildVCard() {
  let card = 'BEGIN:VCARD\r\nVERSION:3.0\r\n';
  card += `N:${last};${first};;;\r\n`;
  card += `FN:${fullName}\r\n`;
  if (phone) card += `TEL;TYPE=CELL:${phone}\r\n`;
  // ... остальные поля
  card += 'END:VCARD';
  return card;
}
```

Разделитель строк `\r\n` обязателен по стандарту RFC 6350.

### Скачивание PNG

```javascript
const link = document.createElement('a');
link.download = 'contact-qr.png';
link.href = canvas.toDataURL('image/png');
link.click();
```

## Кастомизация

### Изменить акцентный цвет интерфейса

```css
:root {
  --accent: #e8ff47;   /* жёлто-зелёный → замени на любой */
}
```

### Добавить поле «Telegram» в vCard

vCard 3.0 не имеет стандартного поля для мессенджеров, но можно использовать `X-` расширение:

```javascript
if (telegram) card += `X-SOCIALPROFILE;TYPE=telegram:${telegram}\r\n`;
```

iOS и большинство приложений игнорируют незнакомые `X-` поля — контакт всё равно сохранится.

## Ограничения

- **Нет SVG-экспорта** — только PNG через Canvas. Для SVG нужна другая библиотека.
- **Нет фото контакта** — vCard поддерживает поле `PHOTO`, но base64-изображение сильно увеличивает QR и ухудшает читаемость.
- **Нет истории** — данные не сохраняются между сессиями.
- **Офлайн-шрифты** — Google Fonts требуют сети при первом открытии; функциональность не страдает.
