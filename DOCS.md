## Обзор проекта

**QR Generator** — это однофайловое веб-приложение для генерации QR-кодов прямо в браузере. Никакого серверного бэкенда, никакой сборки, никакого Node.js. Просто открой `index.html` — и работай.

Проект подходит для:
- быстрого создания QR-кодов для ссылок, текстов, контактов
- встраивания в локальные инструменты и интранет-порталы
- обучения и демонстрации работы с Canvas API

## Быстрый старт

### Вариант 1 — Локально

```bash
git clone https://github.com/stazium-oss/qr-generator.git
cd qr-generator
# Просто открой файл в браузере:
```

3. Сохрани — через минуту сайт будет доступен по адресу:  
   `https://stazium-oss.github.io/qr-generator/`

### Вариант 3 — Любой статический хостинг

Загрузи `index.html` на любой хостинг: Netlify, Vercel, Cloudflare Pages, и т.д.

## Параметры генерации

### Размер (`width` / `height`)

Размер итогового изображения в пикселях. Больший размер — лучше читается на печати.

| Значение | Рекомендуется для |
|---|---|
| 128px | Иконки, превью |
| 200px | Экран, небольшие макеты |
| 256px | Стандартное использование |
| 400px | Печать, плакаты |

### Уровень коррекции ошибок (`correctLevel`)

Определяет, какой процент данных может быть повреждён, и QR всё равно будет прочитан. Высокий уровень = больший QR-код, но надёжнее.

| Уровень | Восстановление | Когда использовать |
|---|---|---|
| **L** | ~7% | Цифровые экраны, идеальные условия |
| **M** | ~15% | Общий случай (по умолчанию) |
| **Q** | ~25% | Частично перекрытые коды, промо-материалы |
| **H** | ~30% | Печать, нанесение на поверхности, логотип поверх QR |

### Цвета (`colorDark` / `colorLight`)

- `colorDark` — цвет тёмных модулей (сам QR-узор). По умолчанию `#000000`.
- `colorLight` — цвет фона. По умолчанию `#ffffff`.

> ⚠️ **Важно:** сканеры QR требуют достаточного контраста между `colorDark` и `colorLight`. Светлый код на тёмном фоне работает хуже, чем тёмный на светлом. Если нужен «инвертированный» QR — выбирай уровень коррекции H.

## Как это работает

### Библиотека qrcodejs

Генерация выполняется через библиотеку [qrcodejs](https://github.com/davidshimjs/qrcodejs), подключённую с CDN:

```html
<script src="https://cdnjs.cloudflare.com/ajax/libs/qrcodejs/1.0.0/qrcode.min.js"></script>
```

Библиотека рисует QR в элемент `<canvas>` внутри целевого `div`.

### Генерация QR (JS)

```javascript
qr = new QRCode(document.getElementById('qr-output'), {
  text: 'https://example.com',  // данные для кодирования
  width: 256,                   // ширина в px
  height: 256,                  // высота в px
  colorDark: '#000000',         // цвет модулей
  colorLight: '#ffffff',        // цвет фона
  correctLevel: QRCode.CorrectLevel.M, // уровень коррекции
});
```

### Скачивание PNG

После генерации `<canvas>` содержит растровое изображение. Скачивание работает через `toDataURL`:

```javascript
const canvas = document.querySelector('#qr-output canvas');
const link = document.createElement('a');
link.download = 'qrcode.png';
link.href = canvas.toDataURL('image/png');
link.click();
```

## Кастомизация

### Изменить размеры по умолчанию

В `index.html` найди `<select id="qr-size">` и измени `option`:

```html
<select id="qr-size">
  <option value="512">512 × 512</option>
  <option value="1024">1024 × 1024</option>
</select>
```

### Изменить цветовую схему интерфейса

Все цвета вынесены в CSS-переменные в начале `<style>`:

```css
:root {
  --bg:      #0a0a0a;   /* фон страницы */
  --surface: #111111;   /* фон карточек */
  --border:  #2a2a2a;   /* цвет рамок */
  --accent:  #e8ff47;   /* акцентный цвет (жёлто-зелёный) */
  --accent2: #ff4757;   /* цвет ошибок */
  --text:    #f0f0f0;   /* основной текст */
  --muted:   #666;      /* второстепенный текст */
}
```

### Добавить имя файла при скачивании

В функции `downloadQR()` измени `link.download`:

```javascript
link.download = `qr-${Date.now()}.png`; // уникальное имя с меткой времени
```

## Ограничения

- **Нет SVG-экспорта** — qrcodejs генерирует только Canvas/PNG. Для SVG потребуется другая библиотека (например, `qrcode-svg`).
- **Нет логотипа поверх QR** — наложение изображения в центр требует дополнительного кода. При необходимости — используй уровень коррекции **H**.
- **Нет истории** — сессия не сохраняется, всё сбрасывается при перезагрузке страницы.
- **Офлайн-режим** — шрифты (Google Fonts) загружаются из сети. При первом открытии без интернета — используются системные шрифты, функциональность не страдает.
